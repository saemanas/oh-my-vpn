# UI Design: Oh My VPN

## 1. Design Philosophy

### A. Principles

| # | Principle | UI Rule |
| --- | --- | --- |
| 1 | **One View, One Action** | One core action per popover view. No multi-step complexity in a single screen |
| 2 | **Numbers Speak** | IP, elapsed time, cost -- 3 numbers define session state. SF Mono for all metrics |
| 3 | **Glass, Not Chrome** | SVG refraction + minimal blur. No decoration -- shape and transparency define boundaries |
| 4 | **Quiet Presence** | While connected, only the menu bar icon. No unnecessary alerts or interruptions |
| 5 | **Destruction = Assurance** | Server destruction is reassurance, not danger. Emphasize "zero trace" on completion |

### B. Density Decision

**Balanced** -- 8px base unit. Generous spacing without waste, matching AlDente's utility app density.

### C. Brand Expression

**Color + Motion composite** -- colorless glass is the brand identity ("freedom = no color"). Liquid Glass spring motion provides macOS native integration.

### D. Anti-Patterns

- Complex dashboards / excessive information display
- Page transitions inside the popover (deep navigation)
- Excessive visual effects (utility apps should feel like tools)
- Dark/hacker tone (harms accessibility and trust)
- Color-only information (always icon + text together)

## 3. Layout System

### A. Popover Structure

| Property | Value |
| --- | --- |
| Width | `320px` fixed |
| Min height | Content-driven |
| Max height | Screen height - menu bar - margin |
| Internal padding | `--space-6` (24px) |
| Section gap | `--space-4` (16px) |
| Navigation | Stack-based push/pop (spring horizontal slide) |
| Overflow | Scroll (region list, etc.) |

### B. Zone Layout

**Disconnected**: Control zone only -- provider selector, region list (scrollable), connect button.

**Connected**: Status zone (badge, session card) + control zone (disconnect button).

### C. Popover Navigation

iOS-style stack push/pop. Back button at top-left. Spring easing horizontal slide (`650ms`). Used for: region detail, provider management, settings, onboarding steps.

## 4. Component Specification

### A. Button

**Purpose**: Primary action trigger across all views.

**Variants**:

| Variant | Tint | Shine | Text Color | Use Case |
| --- | --- | --- | --- | --- |
| success | `--color-success-tint` | success shine | `--color-success-text` | Connect |
| error | `--color-error-tint` | error shine | `--color-error-text` | Disconnect, Destroy |
| warning | `--color-warning-tint` | warning shine | `--color-warning-text` | Loading actions |
| neutral | `--color-tint` | default shine | `--color-text-secondary` | Cancel, secondary |
| info | `--color-info-tint` | info shine | `--color-info-text` | Onboarding CTAs |

**States**: default, hover (scale 1.02 + padding expand), active (scale 0.97), disabled (opacity 0.4), loading (spinner + warning tint).

**Structure**: Liquid Glass 4-layer sandwich (`liquidGlass-wrapper/effect/tint/shine/text`). Border radius `8px`.

### B. Provisioning Stepper

**Purpose**: Provide sense of control during 3-step provisioning.

**Steps**: 1. Creating server → 2. Installing WireGuard → 3. Connecting tunnel.

**Layout**: Vertical list with connector lines between steps.

**Step states**:

| State | Circle | Title | Description |
| --- | --- | --- | --- |
| done | ✓ green tint | Normal weight | "Completed" |
| active | Spinner amber tint | Normal weight | "In progress..." |
| waiting | Number gray | Tertiary color | -- |
| failed | ✕ red tint | Normal weight | Error message inline |

### C. Session Info Panel (Session Card)

**Purpose**: Display key metrics during active connection.

**Structure**: Inner Liquid Glass card with success tint (0.10 opacity).

**Content**:

- IP address: `SF Mono 22px semibold`
- Region: flag emoji + name + provider, `13px`
- Divider
- Metrics row: elapsed time (`⏱`) + cost (`$`), `SF Mono 13px`, space-between

### D. Region List

**Purpose**: Select VPN server region.

**Structure**: Scrollable list inside popover. Each row: flag emoji + region name + hourly cost (mono, right-aligned).

**States**: default, hover (glass tint highlight), selected (last-used region pre-selected).

### E. Input (API Key)

**Purpose**: API key entry during onboarding.

**Structure**: Glass surface input field.

**States**: default, focus (enhanced shine), error (red tint + inline error below), success (green check icon), disabled.

### F. Destruction Confirm Dialog

**Purpose**: Prevent accidental disconnect + communicate destruction outcome.

**Structure**: Glass modal over blur overlay (`8px`).

**Content**: "Server will be destroyed. Continue?"

**Buttons**: Cancel (neutral glass) / Destroy (error glass).

### G. Popover Navigation

**Purpose**: Navigate to detail views within popover.

**Pattern**: iOS-style stack push/pop with back button at top-left.

**Transition**: Spring easing horizontal slide (`650ms`).

## 5. Page Wireframes

> **Wireframe convention**: All non-ASCII symbols are replaced with ASCII equivalents to guarantee monospace alignment. In implementation, render the actual Unicode glyphs.
>
> | Wireframe | Implementation | Note |
> | --- | --- | --- |
> | `[DE]`, `[US]`, `[JP]` | 🇩🇪, 🇺🇸, 🇯🇵 | ISO 3166-1 alpha-2 → regional indicator emoji |
> | `*` | `●` (U+25CF) | Status dot |
> | `~` | `⟳` (U+27F3) | Spinner / loading |
> | `v` | `✓` (U+2713) | Done checkmark |
> | `x` | `✕` (U+2715) | Failure cross |
> | `T` | `⏱` (U+23F1) | Timer icon |
> | `<` | `‹` (U+2039) | Back chevron |
> | `>` | `›` (U+203A) | Forward chevron |
> | `(lock icon)` | 🔒 (U+1F512) | Lock emoji |
>
> **Flag emoji mapping**: `String.fromCodePoint(...code.split('').map(c => 0x1F1E6 + c.charCodeAt(0) - 65))`

## 6. Interaction Patterns

### A. Hover and Press

Liquid Glass locked behavior. All interactive elements:

- Hover: `scale(1.02)` + padding expand + radius expand
- Active: `scale(0.97)`
- Transition: `all 400ms cubic-bezier(0.175, 0.885, 0.32, 2.2)`

### B. Focus Indicator (Glow Ring)

```css
.element:focus-visible {
  outline: none;
  box-shadow: 0 0 0 3px rgba(59, 130, 246, 0.4);
}
```

Applied uniformly to all interactive elements. Minimum 3:1 contrast ratio.

### C. Loading States

| Context | Pattern |
| --- | --- |
| Provisioning | Vertical stepper with spinner on active step |
| API key validation | Spinner inside button, input disabled |
| Server destruction | Inline progress text in popover |
| Region list fetch | Skeleton rows (3 placeholder rows with shimmer) |

### D. Error Pattern Matrix

| Error Type | Display | Recovery |
| --- | --- | --- |
| Provisioning failure | Error glass card below stepper | Cancel / Retry buttons |
| API key invalid | Inline error below input (red tint) | Re-enter key |
| Network failure | Error glass card | Retry option |
| Destruction failure | Warning card with provider console link | Manual action guide |

### E. Success Feedback

| Context | Pattern |
| --- | --- |
| VPN connected | Popover state transition + macOS native notification |
| Server destroyed | Popover state transition + "Server destruction confirmed" notification |
| API key validated | Input green check + success tint + "Provider connected" text |

### F. Transition Inventory

| Transition | Duration + Easing |
| --- | --- |
| Popover entry | `fadeSlideIn 400ms var(--easing-smooth)` |
| Stack push | Horizontal slide right `650ms var(--easing-spring)` |
| Stack pop | Horizontal slide left `650ms var(--easing-spring)` |
| Modal overlay | `fadeIn 200ms var(--easing-smooth)` + blur 8px |
| Toast notification | `slideDown 400ms var(--easing-spring)` |
| State transitions | `crossFade 400ms var(--easing-smooth)` |

### G. Reduced Motion

When `prefers-reduced-motion: reduce`:

- All animations → `fadeIn 200ms linear`
- Spring easing → `ease`
- All durations capped at `200ms`
- Blur effects remain (no motion sensitivity), only movement removed

- Tint opacity → 0.92 (near-opaque)
- SVG displacement scale → 0
- Shine replaced with `1px solid` border
- Full readability for vision-impaired users
